# Руководство по использованию подсистемы сада (обновлено)

## 1. Общее описание
- **Pot** – контейнер, в котором может находиться один объект `Flower`. Реализует `IDraggable` и перемещается игроком.
- **Flower** – отвечает за стадии роста, таймеры ухода и реагирует на полив, удобрение и солнечный свет.
- **GrowthConditions** – `ScriptableObject`, задаёт параметры времени для каждого вида цветов.
- **DragAndDropSystem** – обрабатывает ввод мыши для перетаскивания горшков и отображает визуальную подсказку допустимых зон.
- **FlowerCareSystem** – синглтон, отслеживает все цветы и обеспечивает визуальную подсказку (зеленый/красный индикатор) для допустимых/недопустимых зон размещения.
- **ZoneSelector** – назначает допустимые позиции привязки для каждого `Pot` в своей иерархии.

## 2. Необходимые GameObject‑ы
1. **GardenZoneManager** (пустой GameObject)
   - Добавьте компонент `ZoneSelector`.
   - Добавьте дочерние пустые объекты‑точки привязки (`SnapPoint_01`, `SnapPoint_02` …).
   - Перетащите их в массив **Snap Points** в инспекторе `ZoneSelector`.
2. **Pot Prefab**
   - Добавьте `SpriteRenderer` (спрайт горшка) и скрипт `Pot`.
   - Обязательно `Collider2D` (например, `BoxCollider2D`). поставить галочку на против тригера
   - Поле **Allowed Positions** теперь сериализовано (`[SerializeField] private Vector2[] _allowedPositions;`).
   - Добавьте поле **Flower Prefab** – префаб цветка, который будет инстанцирован из магазина.
3. **Flower Prefab**
   - Добавьте `SpriteRenderer` и скрипт `Flower`.
   - Присвойте ассет `GrowthConditions`.
   - **Stage Sprites** – массив спрайтов, каждый из которых отвечает за конкретную стадию роста (семя, росток, молодой побег, цветок и т.д.).
4. **FlowerCareSystem**
   - Пустой GameObject `FlowerCareSystem`.
   - Присвойте скрипт `FlowerCareSystem`.
   - В дочерний объект `DropZoneHighlight` задайте любой квадратный спрайт (можно белый).
   - Цвета подсказки (`_validColor`, `_invalidColor`) теперь отмечены `[SerializeField]` и могут изменяться в инспекторе.
5. **DragAndDropSystem**
   - Пустой GameObject `DragAndDropSystem` (или разместите скрипт на существующем менеджере).
   - Присвойте скрипт `DragAndDropSystem`.
6. **Shop** (новый объект)
   - Пустой GameObject `Shop`.
   - Присвойте скрипт `Shop`.
   - В инспекторе заполните список **Seed Items** ассетами `SeedItem` (см. пункт 7).

## 3. Создание условий роста (Growth Conditions)
1. **Project** → `Create → Garden → Growth Conditions`.
2. Заполните параметры: время между поливом, удобрением, светом и длительность стадий (`TimeToSprout`, `TimeToYoungShoot`, `TimeToBloom`).
3. Присвойте ассет полю **Conditions** у префаба `Flower`.

## 4. Создание семян (SeedItem)
1. **Project** → `Create → Garden → Seed Item`.
2. **Flower Prefab** – укажите префаб цветка, который будет создаваться при покупке.
3. **Seed Sprite** – (опционально) спрайт, отображаемый в UI магазина.
4. **Price** – стоимость семени в игровой валюте.
5. Сохраните ассет и добавьте его в массив **Seed Items** компонента `Shop`.

## 5. Связка компонентов
- `GardenZoneManager` (с `ZoneSelector`) является родителем всех `Pot` в сцене.
- `ZoneSelector.Awake` заполняет массив `AllowedPositions` у каждого горшка позициями дочерних точек привязки.
- `FlowerCareSystem` создаёт одиночный объект `DropZoneHighlight` и управляет его цветом через сериализованные поля.
- `DragAndDropSystem` вызывает `FlowerCareSystem.Instance.ShowDropZoneFeedback` во время перетаскивания, показывая полупрозрачный квадрат: зеленый — размещение допустимо, красный — недопустимо.
- При отпускании мыши горшок «привязывается» к ближайшей позиции, подсказка скрывается.

## 6. Как работает магазин
1. Игрок нажимает кнопку UI, связанную с `Shop.PurchaseSeed(int index)`.
2. Метод проверяет индекс, снимает валюту (добавьте ваш код) и **инстанцирует** префаб цветка из выбранного `SeedItem`.
3. Созданный объект‑цветок появляется в сцене.
4. Игрок перетаскивает цветок к свободному горшку и отпускает кнопку мыши.
5. `Pot.PlantFlower(Flower flower)` проверяет, что горшок пуст, привязывает цветок к себе и начинает его рост.

## 7. Приём спрайтов для разных стадий роста (внутри Flower)
```csharp
[SerializeField] private Sprite[] stageSprites; // 0‑seed, 1‑sprout, 2‑young shoot, 3‑bloom …
private SpriteRenderer _sr;

private void Awake()
{
    _sr = GetComponent<SpriteRenderer>();
    if (stageSprites != null && stageSprites.Length > 0)
        _sr.sprite = stageSprites[0];
}
...
public void AdvanceToNextStage()
{
    CurrentStage = CurrentStage.NextStage();
    int stageIndex = /* определите индекс из CurrentStage */;
    if (stageSprites != null && stageIndex < stageSprites.Length)
        _sr.sprite = stageSprites[stageIndex];
}
```
*В инспекторе у префаба `Flower` задайте массив `Stage Sprites` — каждый элемент будет использоваться для соответствующей стадии роста.*

## 8. Запуск сцены
1. Нажмите **Play** в Unity.
2. Перетащите любой горшок — появится зеленая/красная подсказка.
3. Откройте UI‑магазин, нажмите кнопку покупки семени — получите цветок.
4. Перетащите цветок в свободный горшок, отпустите кнопку — цветок будет «посажен».
5. Цветок будет проходить стадии роста, меняя спрайт в зависимости от таймеров `GrowthConditions`.

## 9. Расширение системы
- **Новые стадии роста** – создавайте новые классы, реализующие `IGrowthStage`, и обновляйте логику `AdvanceToNextStage`.
- **Разные правила зон** – модифицируйте `ZoneSelector`, чтобы у разных горшков были свои наборы точек.
- **Кастомные визуальные подсказки** – замените квадратный спрайт `DropZoneHighlight` на более сложный (частицы, свечение) и настройте его цвета через инспектор.
